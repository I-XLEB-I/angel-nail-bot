## 2. Быстрые команды (Bot Commands Menu)

Telegram поддерживает команды через `BotFather` — они появляются при нажатии `/` в поле ввода и в меню-иконке слева от него.

### Клиентские команды

Установить через `bot.set_my_commands([...])` при старте (область `BotCommandScopeAllPrivateChats`, исключая админов):

| Команда | Описание |
|---|---|
| `/start` | Главное меню |
| `/book` | Записаться |
| `/mybookings` | Мои записи |

Итого 3 — минимально, ничего лишнего. `/book` и `/mybookings` должны работать из любого состояния, сбрасывая текущий FSM.

### Админские команды

Отдельный набор через `BotCommandScopeChat` для каждого admin `tg_id`:

| Команда | Описание |
|---|---|
| `/start` | Главное меню |
| `/schedule` | Расписание |
| `/requests` | Запросы |
| `/clients` | Клиенты |
| `/admin` | Режим администратора |

### Важное поведение

Любая команда (`/start`, `/book`, `/mybookings`, `/schedule` и т.д.) должна **сбрасывать текущий FSM** и выходить из любого активного потока — онбординга, записи, редактирования шаблона. Пользователь никогда не должен застрять.

Это решает сразу несколько текущих проблем:
- застревание в режиме редактирования шаблона
- неотзывчивость бота при нажатии кнопок клавиатуры во время ввода

## 3. Шаблоны — полный редизайн раздела

### Текущая проблема

Раздел "Шаблоны" показывает все записи из БД одним списком без контекста:
- непонятно, когда каждый шаблон отправляется
- нет категорий
- нет кнопки отмены при редактировании
- если во время редактирования нажать кнопку меню — команда пишется как текст шаблона
- после редактирования чат засоряется сообщениями

### Как должно работать

#### 3.1 Группировка по категориям

Вместо плоского списка — меню категорий:

```
📝 Шаблоны

Выбери раздел:
[💌 Клиентам]
[🏠 Адрес и навигация]
[🗓 Расписание]
[🌴 Другое]
```

При нажатии на категорию — список шаблонов внутри неё с описанием когда отправляется:

**Категория "💌 Клиентам":**

| Ключ шаблона | Название | Когда отправляется |
|---|---|---|
| `booking_confirm` | ✅ Подтверждение записи | Сразу после записи клиентки |
| `reminder_24h` | 🔔 Напоминание за сутки | За 24ч до визита |
| `reminder_2h` | ⏰ Напоминание за 2 часа | За 2ч до визита |
| `postvisit` | 🌷 После визита | Через N часов после визита |
| `repeat_prompt` | 🔄 Приглашение повторить | Через N недель после визита |

**Категория "🏠 Адрес и навигация":**

| Ключ | Название | Когда |
|---|---|---|
| `navigation` | 📍 Адрес (после записи) | В сообщении сразу после подтверждения |
| `navigation_public` | 📍 Адрес (публичный) | В разделе "Адрес и как добраться" |
| `address_post_confirm` | 📍 Адрес + навигация | Полный текст с кодами и этажом |

**Категория "🗓 Расписание":**

| Ключ | Название | Когда |
|---|---|---|
| `schedule_image_header` | Заголовок картинки | Текст над расписанием-картинкой |
| `schedule_image_footer` | Подпись картинки | Текст под расписанием-картинкой |
| `schedule_image_text_override` | Текст на картинке | Если пустой — генерируется автоматически из слотов |

**Категория "🌴 Другое":**

| Ключ | Название | Когда |
|---|---|---|
| `greeting_header` | Приветствие | Главная страница бота |
| `vacation_notice` | Отпуск | Если включён режим отпуска |
| `rules` | Правила визита | В конце подтверждения записи |
| `portfolio_intro` | Портфолио | Раздел "Портфолио работ" |

#### 3.2 Экран редактирования одного шаблона

При нажатии на шаблон показывается:

```
✏️ Подтверждение записи

Когда отправляется:
Сразу после записи клиентки

Текущий текст:
Записала тебя 🪄

24 апреля, 15:00

...
```

Кнопка "Отмена" возвращает к списку категории без сохранения.

#### 3.3 Доступные переменные в шаблонах

Для шаблонов клиентам нужна поддержка переменных-подстановок:

| Переменная | Значение |

| `{name}` | Имя клиентки |

| `{time}` | Время визита (15:00) |
| `{service}` | Название основной услуги |
| `{address}` | Шаблон адреса |

В интерфейсе редактирования показывать список доступных переменных.

## 4. Баг: карточка клиентки не открывается из новой записи

**Симптом:** при нажатии "👀 Открыть карточку клиентки" в уведомлении о новой записи — бот отвечает "Не нашла эту клиентку."

**Причина (из аудита):** `admin/clients.py` — пустой роутер, не подключён в `app.py`. Вся логика карточки клиента — заглушка.

**Что нужно реализовать:**

1. Роутер `admin_clients` подключить в `app.py`
2. Хендлер `admin_clients:open:{user_id}` — открывает карточку клиентки по `user_id`
3. Карточка содержит:
   - Имя, @username, телефон
   - Уровень (логика уровней — см. п. 13)
   - Количество визитов, средний чек, сумму трат
   - Любимую услугу (самая частая в bookings)
   - Дату последнего визита
   - Дату следующей записи (если есть)
   - Заметку мастера (поле `User.note`)
4. Под карточкой кнопки: `✏️ Изменить заметку`, `✉️ Написать`, `🚫 Заблокировать` / `✅ Разблокировать`

## 6. Баг: нет отмены при редактировании шаблона

**Симптом:** бот ждёт текст нового шаблона. Если нажать любую кнопку меню (Клиенты, Настройки и т.д.) — текст кнопки становится значением шаблона.

**Исправления:**

1. Добавить инлайн-кнопку `⬅️ Отмена` под сообщением "Пришли новый текст" — она отменяет FSM и возвращает в список шаблонов без сохранения.

2. Если пользователь присылает текст начинающийся с `/` (команда) — **не сохранять**, сбросить FSM и выполнить команду. Команда важнее редактирования.

3. Любая кнопка ReplyKeyboard в режиме ожидания шаблона — тоже сбрасывает FSM, выполняет соответствующее действие.

## 7. Засорение чата — стратегия "мастер-окна"

**Проблема:** каждое действие порождает новое сообщение. Чат быстро засоряется.

**Решение:** для всех административных потоков использовать `edit_message_text` вместо `send_message`, где возможно.

**Применить к:**

| Раздел | Текущее поведение | Желаемое |
|---|---|---|
| Шаблоны | Новое сообщение на каждый шаг | Одно сообщение редактируется |
| Услуги (редактирование) | Новые сообщения | Одно сообщение редактируется |
| Настройки | Новые сообщения | Одно сообщение редактируется |
| Расписание (добавление слотов) | Новые сообщения | Превью-сообщение редактируется, старые удаляются |
| Рассылка | Новые сообщения | Одно превью-окно |

**Технически:** при входе в раздел бот отправляет одно сообщение и сохраняет его `message_id` в FSM-контексте. Все последующие шаги — `edit_message_text(message_id=...)`. При выходе из раздела сообщение остаётся (не удаляется) — это финальное состояние раздела.

## 8. Рассылка — что это и как работает

**Текущее состояние:** кнопка есть, роутер пустой, пользователь не понимает что это.

**Что нужно реализовать и объяснить:**

Рассылка — отправка одного сообщения всем клиентам у которых есть хотя бы одна запись (статус `confirmed` или `completed`).

**Флоу:**
1. Ангела нажимает "✉️ Рассылка"
2. Бот показывает: "Пришли текст рассылки. Поддерживается Telegram MarkdownV2. Сообщение получат N клиенток."
3. Ангела присылает текст
4. Бот показывает превью + "Отправить всем (N чел.)" → SUCCESS / "Отмена" → DANGER
5. После отправки: "Рассылка отправлена. Доставлено: X из N"

**Важно:** не спамить заблокированным пользователям. Обрабатывать `TelegramForbiddenError` — пометить пользователя как `is_blocked=True` автоматически.

## 9. Плейсхолдеры "На месяц" и "Заблокировать период"

**Текущее:** нажимаешь — вылезает алерт "Этот раздел подключу в одной из следующих админских фаз ✨". Это смущает.

**Решение:** пока функция не реализована — **убрать кнопки из меню**. Не показывать то, что не работает. Когда функция будет готова — добавить обратно.

Альтернатива если хочется оставить кнопки: сделать их серыми/неактивными через callback с `answer(show_alert=False)` и просто ничего не делать.

## 10. Баг: картинка расписания не показывается сразу

**Симптом:** нажимаешь "🖼 Картинка" в расписании → приходит только текст с кнопками, без самого изображения. Нужно дополнительно нажать "👀 Превью".

**Исправление:** при открытии раздела "Картинка расписания" — **сразу генерировать и отправлять изображение**, без необходимости нажимать Превью. Превью оставить как кнопку для принудительного обновления / регенерации картинки.

## 11. Баг: наложение фона при загрузке нового

**Симптом:** при загрузке нового фона он накладывается поверх старого странным образом.

**Что проверить:**
- Сохранение нового фона должно **заменять** старый файл (overwrite), а не добавлять новый
- При генерации картинки расписания использовать только последний загруженный фон
- Путь к файлу фона должен быть фиксированным (например `assets/schedule_bg.jpg`) — чтобы новый файл всегда перезаписывал старый

## 13. Карточка клиентки — редизайн

### Текущая карточка (что есть)
- Имя, @username
- Уровень ("НОВЕНЬКАЯ" / уровни)
- Визитов, средний чек, отмен, потрачено
- Любимая услуга, последний визит, последние даты, телефон

### Что оставить, что убрать, что добавить

**Оставить:**
- Имя + @username
- Уровень + подпись уровня
- Визитов (количество завершённых)
- Потрачено (сумма completed bookings с фиксированной ценой)
- Любимая услуга
- Последний визит (дата)
- Телефон

**Убрать:**
- "Средний чек" — лишнее, Ангела не использует
- "Отмен" — спорная метрика, скорее смущает чем помогает. Если оставить — переименовать в "Отменила записей" и показывать только если > 0

**Добавить:**
- Следующая запись (дата + время) — если есть активная
- Заметка мастера (поле `User.note`) — отображать если заполнена

### Система уровней (предложение)

| Уровень | Название | Условие |
|---|---|---|
| 1 | Новенькая | 0 визитов |
| 2 | Постоянная | 3–9 визитов |
| 3 | Своя девочка | 10–19 визитов |
| 4 | Любимица | 20+ визитов |

Подпись под уровнем:
- Новенькая → "Самое красивое ещё впереди"
- Постоянная → "Уже знает как будет красиво"  
- Своя девочка → "Проверенная, любимая"
- Любимица → "Самая преданная"
