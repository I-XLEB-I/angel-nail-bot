# Dynamic Image Style Guide

Обновлённые динамические изображения в боте собираются по одной системе, а не по отдельным
локальным шаблонам. Базовая логика живёт в `src/services/image_theme.py`.

## Направление

- Визуальный тон: `champagne / prism / airy luxury`
- База: светлый rose-beige градиент с мягкими prism-flare акцентами
- Настроение: чисто, дорого, воздушно, без тяжёлого dark-glass оформления
- Источник направления: существующие референсы из `assets/reference_layouts`

## Обязательные правила

- Любая динамическая картинка использует общий фон через `load_theme_background(...)`
- Верхний брендовый lockup всегда строится через `draw_angels_brand(...)`
- Главный заголовок всегда строится через `draw_section_title(...)` или `draw_header_serif(...)`
- Основной контент всегда помещается в одну общую frosted-панель через `draw_glass_panel(...)`
- Локальные теги, дни, метрики и статусы оформляются только через `draw_soft_badge(...)`
- Нижний брендовый ритм держится через `draw_accent_divider(...)` и `draw_footer_brand(...)`

## Типографика

- Крупные брендовые заголовки: serif (`FONT_CANDIDATES_SERIF`)
- Мягкие подписи и романтизирующие акценты: script (`FONT_CANDIDATES_SCRIPT`)
- Рабочий контент, цифры, списки, статусы: body (`FONT_CANDIDATES_BODY`)
- На фоне используется светлый hero-текст
- Внутри панелей используется тёплый тёмный `ink`, а не белый текст

## Композиция

- Верх: бренд, затем один главный заголовок, затем тонкий divider
- Центр: одна главная панель для полезного контента
- Низ: note/divider/footer без дублирования заголовка
- Воздух важнее плотности: не забивать панель текстом до краёв

## Поведение контента

- Длинные строки режутся или переносятся до потери читабельности, а не ужимаются бесконечно
- Overflow обозначается явно (`+ ЕЩЁ ...` / `СМ. В БОТЕ`)
- Отменённые записи остаются видимыми, но уходят в приглушённый тон
- Если футер шаблона совпадает с брендом `ANGELS NAIL SPACE`, повторно как note он не рисуется

## Где применяется

- `src/services/schedule_image.py`
- `src/services/price_image.py`
- `src/services/bookings_image.py`
- `src/services/client_card_image.py`
- `src/services/section_image.py`
